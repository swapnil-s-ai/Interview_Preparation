# Guiding Principles

This repository is built to maximize interview readiness, not to create the largest collection of notes.

---

## Learning Principles

1. Interview Preparation > Documentation *Interview-first mindset.*
2. Learn → Practice → Explain → Document → Commit
3. One primary resource per topic.
4. Every concept must connect to a real project whenever possible.
5. Theory without coding is incomplete.
6. Coding without explanation is incomplete.
7. Weekly revision is mandatory.
8. Daily consistency is more valuable than occasional long study sessions.
9. AI is a reviewer, not the primary problem solver.

---

# Documentation Standards

## Philosophy

- High Signal > High Volume
- Learning > Documentation
- Interview First
- One File = One Responsibility
- No Duplicate Content
- Repository grows with learning.
- Every paragraph must provide interview value.

---

Every content file begins with:

## File Responsibilities

| File | Purpose |
|------|---------|
| Theory | Understand concepts |
| Syntax | Quick syntax reference |
| Practice | Coding questions & mistakes |
| Interview FAQ | Frequently asked interview questions |
| Cheat Sheet | 5-minute revision notes |
| Assessment | Progress evaluation |
| Revision | Revision history & weak topics |
| Project Mapping | Connect concepts to real work |

| Assessment | 1–2 pages |
| Revision | Living document |

---

## Quality Checklist

Before committing any content, ask:

- Does this improve interview readiness?
- Is this concise?
- Is duplicate information avoided?
- Can I revise this quickly?
- Would I read this one week before an interview?

If the answer to any question is **No**, improve or remove the content.

---

## Repository Rule

The repository supports learning.

The repository must never become the reason learning slows down.
